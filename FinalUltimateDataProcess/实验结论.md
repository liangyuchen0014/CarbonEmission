# 模型选择
是的，您的直觉非常敏锐且符合物理规律（例如内燃机的万有特性曲线通常是非线性的）。单纯的线性回归（$y = w_1x_1 + w_2x_2 + b$）无法捕捉这种“先增后减”或“U型”的非线性关系。

为了捕捉这种关系，我们不需要立刻跳到深度学习，有几种非常成熟且高效的方法可以解决这个问题：

### 1. 多项式回归 (Polynomial Regression)
这是最直接的方法。我们不只使用 $Speed$ 和 $Power$，而是显式地构造它们的**高次项**和**交互项**作为新特征输入给线性模型。

*   **原理**：将输入特征扩展为 $[Speed, Power, Speed^2, Power^2, Speed \times Power]$。
*   **效果**：模型变成了 $y = w_1 S + w_2 P + w_3 S^2 + w_4 P^2 + w_5 SP + b$。这样就能拟合抛物线（二次函数）了。
*   **实现**：在 Scikit-Learn 中使用 `PolynomialFeatures`。

### 2. 广义加性模型 (GAMs - Generalized Additive Models)
这是一个比多项式回归更灵活、比随机森林更可解释的强大统计模型。

*   **原理**：它假设 $y = f_1(Speed) + f_2(Power) + b$，其中 $f_1$ 和 $f_2$ 不是简单的直线，而是**平滑样条函数 (Splines)**。
*   **优点**：它可以自动学习出 Speed 对油耗的非线性曲线（比如波浪形、U形），同时保持极高的可解释性（你可以画出 $f_1$ 的曲线图）。
*   **库**：Python 中可以使用 `pygam` 库。

### 3. 现有的树模型 (Random Forest / XGBoost)
您代码中已经支持的 RF、XGBoost 和 LightGBM **天生就具备拟合非线性的能力**。
*   它们通过将空间切割成很多小块（分段常数近似），可以逼近任意复杂的非线性函数。如果数据量足够，它们通常能很好地捕捉这种关系。

关于指标是否归一化：

MAE (0.0177) 和 RMSE (0.0229) 是绝对指标，它们没有归一化，单位与您的目标变量 usage_rate 相同。
R2 (0.9413) 是相对指标（归一化的），它表示模型解释了 94.13% 的方差，这在任何尺度下都是一个非常高的分数，说明模型拟合得很好。
关于数值较小 (0.1~0.5) 的影响：

确实，当目标值很小时，绝对误差（如 0.017）看起来很小，但相对于真实值（如 0.1）可能占比不低（17%）。
为了更直观地评估模型在小数值上的表现，引入 MAPE (平均绝对百分比误差) 会更有参考价值。

# 实验结果分析
这段输出揭示了 Speed（速度）和 Power（功率）对油耗（Usage Rate）影响的关键信息。以下是详细解读：

### 1. 变量相关性
*   **`Correlation between Speed and Power: 0.7101`**
    *   **解读**：速度和功率之间存在**中强正相关**（0.71）。这意味着车速越快，通常功率也越高，但两者并非完全同步（相关性未达到 0.9 以上的极高水平）。这说明功率中包含了一些速度无法解释的信息（例如爬坡、负载、加速阻力等）。

### 2. 变量影响力 (Full Model)
*   **`Full Model Coefficients: Speed=-0.0005, Power=0.0032`**
    *   **解读**：
        *   **Power (0.0032)**：系数为正，符合物理常识——功率越大，油耗越高。
        *   **Speed (-0.0005)**：系数为负，这在多元回归中是有可能的（例如在相同功率下，速度越快意味着效率越高/阻力越小？或者仅仅是统计上的调整）。但要注意它的绝对值很小。
*   **`Relative Importance (Standardized): Speed=6.8%, Power=93.2%`**
    *   **解读**：这是最关键的指标。经过标准化处理（消除了单位差异）后，**功率（Power）对油耗变化的解释贡献了 93.2%，而速度（Speed）仅贡献了 6.8%**。
    *   **结论**：**功率是决定油耗的绝对主导因素**。速度虽然相关，但在已知功率的情况下，它对预测油耗的额外帮助非常微弱。

### 3. 模型精度与消融实验
*   **`Full Model Metrics: R2: 0.8948`**
    *   **解读**：全模型（Speed + Power）表现非常出色，能解释约 89.5% 的油耗变化。
*   **`Reduced Model Metrics (Speed Only): R2: 0.4031`**
    *   **解读**：如果移除功率，仅用速度预测，模型能力**断崖式下跌**，$R^2$ 仅剩 0.40。
*   **`Impact of removing Power: R2 dropped by 0.4916`**
    *   **解读**：移除功率变量导致模型解释能力下降了近 50 个百分点。

### 总结论
1.  **Power 是核心**：对于预测油耗，**功率（Power）是不可或缺的核心特征**。它直接反映了发动机的做功情况，与油耗有最直接的物理联系。
2.  **Speed 是次要/辅助**：虽然速度与油耗相关，但这种相关性很大程度上是因为“速度高通常功率也高”。一旦我们掌握了功率数据，速度数据提供的额外信息量就非常有限了（仅占不到 7% 的重要性）。
3.  **不能移除 Power**：如果为了简化模型而移除功率变量，预测精度将无法接受（从优秀的 0.89 降至不及格的 0.40）。

# 外推能力分析
线性回归模型在该任务中的外推能力如何？对样本没覆盖到的区域（如低速或高速、较低的功率和较大的功率）能准确预测吗？

基于代码实现、模型数学特性、实验结果以及物理现实经验，关于**线性回归模型（Linear Regression）**在该任务中的外推能力（Extrapolation Capability），结论如下：

### 结论：外推能力较弱，存在显著的物理一致性风险

虽然线性回归在训练数据覆盖的范围内（内插）表现出了极高的精度（$R^2 \approx 0.94$），但在样本未覆盖的区域（外推），它极有可能给出**违反物理规律**或**偏差巨大**的预测。

以下是详细分析：

#### 1. 数学特性与物理规律的冲突
*   **模型特性**：线性回归假设输入与输出是单调线性的（$y = w_1x_1 + w_2x_2 + b$）。这意味着无论速度或功率增加多少，油耗都按固定比例增加。
*   **物理现实**：
    *   **空气阻力**：现实中，空气阻力与速度的**平方**成正比，克服阻力所需的功率与速度的**立方**成正比。在高流速区域，油耗的增长是非线性的（指数级）。线性模型在高流速/高功率区会显著**低估**油耗。
    *   **万有特性（BSFC）**：内燃机的效率不是常数。在极低功率（如怠速）或极高功率下，燃油效率通常较低（油耗率高）。线性模型无法表达这种“U型”或复杂的效率曲面。

#### 2. 极端区域的具体表现预测

*   **低速/怠速区域 (Low Speed / Idle)**
    *   **预测风险**：**可能出现负值或过低估计**。
    *   **原因**：如果训练数据集中主要包含中高负荷的样本，模型拟合出的截距（Intercept）可能是负数（为了拟合高处的斜率）。当 $Speed \approx 0, Power \approx 0$ 时，模型可能预测出负油耗，这在物理上是不可能的。
    *   **现实**：怠速时速度为0，功率接近0，但油耗是一个固定的正值（维持发动机运转）。

*   **高速/高功率区域 (High Speed / High Power)**
    *   **预测风险**：**显著低估**。
    *   **原因**：如前所述，高速下的风阻导致能耗急剧上升。线性模型只能画直线，无法跟上物理上呈二次或三次函数增长的油耗曲线。

*   **未覆盖的组合区域 (The "Gap" Areas)**
    *   **数据分布**：实验显示 Speed 和 Power 的相关系数为 **0.71**。这意味着数据主要分布在一个对角线带状区域（速度高时功率通常也高）。
    *   **盲区**：模型几乎没见过“**低速高功率**”（如陡坡起步、重载低速）或“**高速低功率**”（如长下坡）的样本。
    *   **预测**：在“低速高功率”区域，线性模型仅根据 Power 的系数 $w_2$ 做加法，忽略了低速重负荷下发动机极差的燃烧效率，导致预测值可能**偏低**。

#### 3. 与其他模型的对比（关于外推）

值得注意的是，虽然线性回归外推能力弱，但它在某种程度上比树模型（Random Forest / XGBoost）“好”一点点：
*   **树模型**：在训练范围之外，树模型只能预测出训练集中最大/最小的那个叶子节点的值（**常数预测**），完全没有外推趋势。
*   **线性模型**：至少还能保持一个增长的趋势，尽管增长的速率（斜率）可能是不准确的。
*   **多项式回归**：虽然能拟合非线性，但在外推区域极易**爆炸**（Runge现象），预测出极其离谱的天文数字。

### 总结建议

如果您的应用场景包含大量未在训练集中出现的工况（例如新路线、极端天气、不同驾驶习惯）：

1.  **不要盲目信任线性模型的外推值**。
2.  **物理约束**：在代码中对预测结果加后处理（Post-processing），例如 `prediction = max(prediction, idle_consumption)`，防止出现负值。
3.  **特征工程**：引入物理特征（如 $Speed^2$, $Speed \times Power$）给线性模型，能稍微改善其符合物理规律的能力，但仍需谨慎。

**对于线性回归模型，给定speed、power，模型输出预测结果的同时，能否输出对该结果的置信度？**

标准的 sklearn.linear_model.LinearRegression 只输出点估计（Point Estimate），即一个具体的数值，默认不包含置信度。

但是，对于线性回归，我们可以通过统计学方法计算预测区间 (Prediction Interval)。

什么是预测区间？
它回答了这个问题：“对于给定的 Speed 和 Power，我有 95% 的把握认为真实的油耗会落在 [下界, 上界] 这个范围内。”
区间的宽度反映了“置信度”：区间越窄，说明模型越确信；区间越宽，说明不确定性越高（通常发生在训练数据稀疏的区域，即外推区域）。

实现方案
我们需要在 fit 阶段记录训练数据的统计特征（协方差矩阵的逆、残差方差），然后在 predict 阶段利用这些特征计算每个预测点的标准误差。

# 二、公式推导
线性回归模型输入speed、power，输出预测值usage_rate。
预测得到的$usage\_rate$的单位是L/min，则可得每小时的油耗/气耗预测值hour_usage_rate为：
$$hour\_usage\_rate = 60*usage\_rate$$
其单位为L/h；同时，该小时的平均车速是自变量$speed\_mean$，是已知的，因此可得百公里油耗/气耗预测值$pred\_consum\_per\_100km$计算公式为
$$pred\_consum\_per\_100km = \frac{hour\_usage\_rate*100}{speed}$$
由于每辆车的实际平均油耗/气耗$act\_consum\_per\_100km$是已知的（见车辆信息表格），因此可得到该车在该速度和功率下的偏移系数λ：
$$ λ = \frac{pred\_consum\_per\_100km}{act\_consum\_per\_100km}$$
